# 企鹅大乱斗 - 产品经理审查报告

> 审查日期: 2026-06-18
> 审查范围: 全部 26 个源文件（15 Vue 组件 + 11 数据文件）
> 代码量: ~2,855 行 / ~108 KB

---

## 一、总体评价

**一句话总结**: 壳子搭得不错，但里面是空的。

游戏的 UI 框架和数据结构设计得相当完整，13 个 Tab 页全部有对应组件，
27 种武器、31 种技能、10 个关卡的数据都定义好了。看起来很丰满。

但实际跑起来，玩家能做的事情只有两件：
1. 点「乐斗」→ 选关卡 → 看自动战斗 → 拿经验
2. 点「斗神塔」→ 打塔 → 拿经验

其他 11 个 Tab 页要么是展示用的，要么功能是断的。

**完成度评估**: 60%

---

## 二、致命问题（必须修）

### 2.1 技能升级完全不能用

**文件**: `SkillBook.vue` 第 112 行

```javascript
const ownedSkill = computed(() => null)  // ← 硬编码返回 null
```

整个技能升级 UI 是废的。玩家看到技能列表，但：
- 已拥有的技能没有 ✓ 标记
- 等级永远显示 Lv.0/5
- 升级按钮永远不会出现
- 进度条永远是 0%

**影响**: 精魄（spirit）这个核心货币没有消费出口。玩家打了 10 局攒了 30 精魄，花不掉。

### 2.2 商城道具买了没用

**文件**: `ShopPanel.vue` + `shop.js`

| 道具 | 问题 |
|------|------|
| 致命一击卷轴 | effect 类型是 `'buff'`，但 `applyEffect()` 不处理 buff 类型 → 花 150 经验买了个寂寞 |
| 强化石 | 是材料，但没有消耗它的系统 |
| 保护券 | 同上 |
| 回春丹 | 用 `state.player.maxHealth` 计算回复量，但这个属性可能不存在 |

### 2.3 职业技能树没有实际效果

**文件**: `ClassTree.vue` + `classes.js` + `engine.js`

玩家可以在 4 个职业的 7 层技能树里分配技能点，UI 交互做得挺好的。

但 `calculateClassStats()` 只处理了 3 种效果（maxHealth、speed、allStats），其他十几种效果（weaponDamage、damageReduce、critRate、dodgeRate、counterRate 等）全部被忽略。

**更关键的是**: 战斗引擎 `engine.js` 根本没有读取职业数据。分配不分配，打起来完全一样。

### 2.4 成就系统永远不会解锁

**文件**: `AchievementPanel.vue` + `achievements.js`

`checkAchievements()` 函数写好了，但没有任何地方调用它。
玩家打完 100 场战斗，成就列表还是全灰的。

### 2.5 境界系统是摆设

**文件**: `RealmPanel.vue` + `realms.js`

- 境界突破按钮可以用，`state.realm.level` 会递增
- 但 `getCurrentRealm()` 用的是玩家等级，不是境界等级
- 境界加成（力量/敏捷/速度/生命）从未应用到战斗计算

## 三、严重问题（应该修）

### 3.1 好友好感度不会增长

`FriendList.vue` 里打了好友，但 `battleFriend()` 函数不更新好感度。
好感度永远是初始值。

### 3.2 排行榜每次渲染都随机

```javascript
streak: Math.floor(Math.random() * 20)  // 每次 re-render 都重新随机
```

排行榜的连胜数据在每次页面刷新时都会变。玩家会觉得「怎么刚才还是第 3 现在第 7 了」。

### 3.3 装备系统没有获取途径

`EquipmentPanel.vue` 的 UI 做好了，16 件装备数据定义了，5 个槽位、2 套套装。

但没有任何地方能获取装备。没有掉落、没有商店、没有合成。
装备面板打开是空的，永远是空的。

### 3.4 武器槽位没有装备数据

`equipment.js` 里定义了头盔、铠甲、靴子、饰品，但武器槽位没有装备数据。
装备面板的武器槽永远显示「未装备」。

### 3.5 NPC 敌人技能是空壳

`pets.js` 里敌人定义了 `skills: ['basic_attack', 'ice_shard', 'frost_breath']`，
但这些字符串在技能数据库里不存在。战斗引擎里敌人只会普攻。

打了 10 个关卡，每个 Boss 都是只会平砍。

### 3.6 体力上限显示错误

`StageSelect.vue` 里体力上限硬编码为 100，但实际应该是 `100 + (level-1)*5`。
Lv.20 的玩家体力上限应该是 195，但显示还是 100。

### 3.7 好友战经验可能为负

```javascript
exp: 45 + (friend.level - state.player.level) * 3
```

如果玩家 Lv.30 打 Lv.10 好友：`45 + (10-30)*3 = 45 - 60 = -15`。
经验变负数。

## 四、体验问题（建议修）

### 4.1 alert() 满天飞

至少 5 个组件用了原生 `alert()` 做用户提示：
- TowerScene（扫荡结果）
- ShopPanel（购买确认）
- RealmPanel（突破确认）
- StageSelect（休息恢复）
- ClassTree（转职确认）
- GameMenu（重置确认）

这在桌面浏览器上勉强能用，移动端体验极差。
需要一个统一的 Toast/Modal 组件。

### 4.2 战斗日志被截断

`BattleScene.vue` 每回合只显示最后 5 条日志，敌方回合只显示最后 3 条。
如果一个回合内有多次连击/反击/反弹，后面的全部被吃掉。

### 4.3 切换战斗速度不能实时生效

改变速度后，已经创建的 Promise 里的 setTimeout 不会更新。
要等到当前动画结束才能生效。

### 4.4 存档导入没有校验

`GameMenu.vue` 的导入功能直接 `Object.assign(state, data)`。
如果导入一个恶意或格式错误的 JSON，整个游戏状态会损坏。
没有版本号校验、没有字段校验、没有类型检查。

### 4.5 代码重复

`LEVEL_RANGES` 和 `getLevelRange()` 在 `skills.js` 和 `weapons.js` 里各写了一遍。
`QUALITY_COLORS` 在 `constants.js` 和 `weapons.js` 里各定义了一份。

## 六、后续方向建议

### 第一优先级：让现有系统能用（1-2 周）

这一步不需要新功能，只需要把断掉的连接接上。

**P0-1. 修复技能升级**
- 把 `ownedSkill` 的 computed 改成真正查找 `state.skills`
- 让升级按钮出现、扣精魄、加等级

**P0-2. 修复商城道具**
- 处理 `buff` 类型效果（至少让暴击卷轴生效）
- 删除或禁用无法使用的道具（强化石、保护券）
- 修复回春丹的 maxHealth 引用

**P0-3. 接入职业系统到战斗引擎**
- `engine.js` 的 `playerTurn()` 和 `enemyTurn()` 里读取职业加成
- 至少让被动技能（减伤、暴击率、闪避率）生效

**P0-4. 接入境界加成**
- 在 `getCombatStats()` 里加上境界 bonus

**P0-5. 接入成就检查**
- 在 `addWin()`、`addExp()`、升级等关键节点调用 `checkAchievements()`

**P0-6. 修复体力上限**
- `STAMINA_MAX` 改成函数：`100 + (level-1) * 5`

**P0-7. 统一 Toast 组件**
- 干掉所有 `alert()`，用一个轻量 Toast 替代

### 第二优先级：补齐核心循环（2-3 周）

**P1-1. NPC 敌人技能系统**
- 给敌人定义真正的技能（不只是字符串引用）
- 让 Boss 会用冰息、眩晕等技能
- 战斗会更有策略性

**P1-2. 装备获取途径**
- 关卡通关有概率掉落装备
- 斗神塔高层掉落高品质装备
- 商城可以购买基础装备

**P1-3. 秘境副本**
- 参考设计文档的 7 级秘境
- 每个秘境有多层 + Boss
- 掉落装备和材料

**P1-4. 好友好感度修复**
- 对战后增加好感度
- 好感度影响经验加成

**P1-5. 连胜/连败状态**
- 连胜 ≥ 3 → 亢奋（攻击 +30%）
- 连败 ≥ 3 → 低迷（攻击 -20%）
- 在战斗引擎里应用

### 第三优先级：内容扩展（3-4 周）

**P2-1. 更多关卡**
- 当前只有 10 关，Lv.50 就通关了
- 扩展到 20-30 关，覆盖到 Lv.100

**P2-2. 每日任务**
- 每日登录奖励
- 每日挑战 X 次
- 每日通关 X 关

**P2-3. 排行榜持久化**
- NPC 的数据固定下来，不要每次随机
- 加入更多 NPC 让排行榜更丰富

**P2-4. 存档校验**
- 导入时校验版本号和必要字段
- 缺失字段用默认值填充
- 类型不匹配时拒绝导入

**P2-5. 战斗日志优化**
- 不截断日志，改用虚拟滚动
- 或者限制每回合最多显示 10 条

## 八、总结

这个游戏的数据设计水平不错 — 武器、技能、关卡的数据结构很完整，
权重概率系统、品质系统、套装系统的设计思路都是对的。

但「设计」和「实现」之间有巨大鸿沟。13 个 Tab 页里，只有 2-3 个是真正能玩的。
其他要么是展示层（境界、成就），要么是半成品（装备、职业），要么是断路的（技能升级、商城道具）。

**核心问题**: 系统之间没有连接。数据定义了，UI 做了，但中间的「胶水逻辑」没有。
技能点分配了 → 战斗不读取。装备定义了 → 没有获取途径。成就写好了 → 没有触发机制。

**建议**: 暂停新功能开发，用 1-2 周时间把现有系统的断路接上。
一个能用的 5 个 Tab 页比 13 个半成品 Tab 页体验好得多。
