> 本文档基于原版设计 + duanwuqiufenmao.top/qpet 逆向分析，供 AI 编码使用。

## 一、游戏概述

**名称**: 企鹅大乱斗  
**类型**: 回合制宠物对战游戏（PVE + PVP）  
**技术栈**: Vue 3 + Vite + Tailwind CSS 4  
**存储**: localStorage 自动存档（纯前端，无后端）  
**入口**: `/#/game`  
**设计目标**: 单人可玩的完整回合制对战游戏，包含角色成长、装备收集、多模式挑战

**随机属性**: 每升一级额外随机获得2点属性（分配到力量/敏捷/速度）

**经验公式**:
```
升级所需经验 = Math.floor(100 * Math.pow(1.15, level - 1))
获得经验 = 基础经验 × 对手等级系数
对手等级系数 = max(10, min(80, 45 + (对手等级 - 我方等级) × 3))
```

### 3. 技能获取机制

**累积池 + 权重概率**:
- 每个等级段有不同的技能池
- 按权重随机抽取新技能
- 未获得的技能会累积权重（每次未抽到 +10% 权重）

**等级段开放节奏**:

| 等级段 | 新增技能 |
|--------|----------|
| Lv.1-4 | 5种属性技能 |
| Lv.5-9 | +5种被动/攻击 |
| Lv.10-14 | +8种防御/控制 |
| Lv.15-19 | +6种高阶技能 |
| Lv.20-24 | +3种神技 |
| Lv.30-34 | +4种终极技能 |

### 4. 技能升级

- 消耗精魄升级
- 每级消耗 = 基础消耗 × 当前等级
- 最高 Lv.5（所有技能统一）

## 五、战斗系统

### 1. 战斗流程

```
开始战斗
    ↓
计算先手（速度高者先行动）
    ↓
轮到行动方：
  1. 检查是否眩晕/封印
  2. AI决策：选择武器 or 技能 or 普攻
  3. 计算伤害（含暴击/连击/格挡/闪避/反击）
  4. 应用伤害 + 特效
  5. 检查装死/重生
    ↓
轮到对方行动
    ↓
检查战斗结束条件（任一方HP≤0）
    ↓
循环或结算
```

### 2. AI技能选择逻辑

```javascript
function chooseAction(character, enemy) {
  // 1. 血量危急时优先回复
  if (character.hp / character.maxHp < 0.3 && hasHealSkill(character)) {
    return useHealSkill(character);
  }
  
  // 2. 敌方血量低时用控制技能收割
  if (enemy.hp / enemy.maxHp < 0.2 && hasControlSkill(character)) {
    return useControlSkill(character);
  }
  
  // 3. 60%概率使用高伤害技能
  if (Math.random() < 0.6 && hasAttackSkill(character)) {
    return useAttackSkill(character);
  }
  
  // 4. 30%概率使用控制技能
  if (Math.random() < 0.3 && hasControlSkill(character)) {
    return useControlSkill(character);
  }
  
  // 5. 默认普攻（使用武器）
  return normalAttack(character);
}
```

### 3. 伤害公式

```
// 基础伤害
baseDamage = str × weaponBaseDamage / 50

// 技能加成
skillDamage = baseDamage × skillMultiplier × (1 + weaponEnhanceBonus)

// 最终伤害
finalDamage = skillDamage × randomFactor(0.9~1.1) × (1 - defenseReduction)

// 暴击
critRate = min(0.05 + agi × 0.002, 0.4)
critDamage = 1.5×  // 暴击时伤害×1.5

// 闪避
dodgeRate = min(0.03 + agi × 0.001 + spd × 0.001, 0.35)

// 格挡
blockRate = 15%  // 可通过技能提升
blockReduction = 50%  // 格挡时减伤50%

// 反击
counterRate = min(0.1 + agi × 0.001, 0.3)
counterDamage = 50%  // 反击造成50%伤害

// 连击
comboRate = weaponComboRate + skillComboRate
comboDamage = 50%  // 连击造成50%伤害

// 眩晕
stunRate = weaponStunRate
stunDuration = 1回合

// 反弹
reflectRate = skillReflectRate
reflectDamage = 受到伤害的X%
```

function buyItem(itemId) {
  const item = shopItems[itemId];
  if (player.exp < item.price) return "经验不足";

## 十八、技术注意事项

### 1. 纯前端实现

- 所有数据存储在 localStorage
- 无需后端API
- PVP模拟为与NPC好友对战（非实时联机）
- 排行榜为本地模拟数据

### 2. 性能优化

- 战斗日志使用虚拟滚动
- 大量数据使用懒加载
- 存档使用压缩JSON

### 3. 代码规范

- 使用 Vue 3 Composition API
- 使用 Pinia 状态管理
- 使用 TypeScript 类型定义
- 组件职责单一，便于AI理解和维护

### 4. AI编码提示词建议

在让AI实现时，建议按以下顺序：
1. 先实现 `game/data/` 下所有数据文件
2. 再实现 `game/engine/battle.js` 战斗引擎
3. 然后实现 `game/store.js` 状态管理
4. 最后逐个实现UI组件
5. 每个系统独立测试后再集成
